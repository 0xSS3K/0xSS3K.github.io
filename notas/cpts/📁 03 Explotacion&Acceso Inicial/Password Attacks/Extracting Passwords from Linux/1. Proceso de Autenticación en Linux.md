## Conceptos Clave (TL;DR)

* Linux utiliza Pluggable Authentication Modules (PAM) para gestionar información de usuarios, autenticación y sesiones.
* El archivo `/etc/passwd` es legible por cualquier usuario y contiene la estructura de las cuentas; el valor `x` indica que el hash está delegado.
* El archivo `/etc/shadow` requiere privilegios administrativos y almacena los hashes de las contraseñas.
* Los hashes siguen la estructura `$<id>$<salt>$<hashed>`, donde el `<id>` revela el algoritmo criptográfico empleado.

  
## Herramientas Clave

* **unshadow**: Utilidad incluida en John the Ripper (JtR) que combina los archivos passwd y shadow en un formato unificado listo para ser atacado.

* **[Hashcat](../../../📂%2008%20Herramientas&Cheatsheets/Hashcat.md)**: Herramienta avanzada para la recuperación y cracking de contraseñas mediante fuerza bruta o diccionario.

* **John the Ripper (JtR)**: Alternativa de cracking que resulta especialmente útil en su modo "single crack" para este tipo de escenarios.

  
## Metodología Paso a Paso

1. **Identificación de vulnerabilidades de configuración**: Comprobar si `/etc/passwd` expone hashes en lugar de una `x` (común en sistemas muy antiguos) o si tiene permisos de escritura accidentales.

2. **Recolección de archivos críticos**: Tras obtener acceso root, realizar copias locales de `/etc/passwd`, `/etc/shadow` y, de ser posible, `/etc/security/opasswd`.

3. **Preparación de los hashes**: Ejecutar `unshadow` para fusionar la información de usuarios y sus hashes correspondientes en un solo archivo analizable.

4. **Cracking Offline**: Someter el archivo combinado a ataques de diccionario o reglas utilizando Hashcat o JtR para recuperar las contraseñas en texto claro.

## Cheat Sheet de Comandos

```bash
# Leer la primera linea de passwd para verificar si existe la 'x' o se puede abusar de la cuenta root

head -n 1 /etc/passwd
```
  
```bash
# Cambiar al usuario root (útil si se logró eliminar la 'x' en passwd indicando login sin contraseña)

su
```
  
```bash
# Leer el historial de contraseñas antiguas para buscar algoritmos débiles o patrones de reuso

sudo cat /etc/security/opasswd
```
  
```bash
# Crear copias de seguridad de los archivos de autenticación antes de la extracción

sudo cp /etc/passwd /tmp/<PASSWD_BACKUP_NAME>
sudo cp /etc/shadow /tmp/<SHADOW_BACKUP_NAME>
```

```bash
# Combinar ambos archivos en un formato reconocible por herramientas de cracking

unshadow /tmp/<PASSWD_BACKUP_NAME> /tmp/<SHADOW_BACKUP_NAME> > /tmp/<UNSHADOWED_FILE_NAME>
```

```bash
# Crackear los hashes usando Hashcat.
# -m 1800: Especifica el tipo de hash (SHA-512 en este ejemplo, cambiar según el ID detectado).
# -a 0: Modo de ataque de diccionario (Straight).
# -o: Archivo de salida para las contraseñas recuperadas.

hashcat -m <HASH_TYPE_ID> -a 0 /tmp/<UNSHADOWED_FILE_NAME> <PATH_TO_WORDLIST> -o /tmp/<CRACKED_OUTPUT_NAME>
```


## "Gotchas" y Troubleshooting

* **Privilege Escalation Instantáneo**: Si `/etc/passwd` es escribible por error, se puede borrar el campo de la contraseña (la `x` y los dos puntos adyacentes) para la cuenta root, lo que permite ejecutar `su` e iniciar sesión sin que se solicite contraseña.

* **Cuentas Restringidas**: Si el campo de contraseña en `/etc/shadow` contiene un `!` o un `*`, el inicio de sesión mediante contraseña Unix tradicional está deshabilitado. Sin embargo, la cuenta aún podría ser accesible mediante llaves de autenticación o Kerberos.

* **Identificación del Hash**: El ID en el hash es clave. `1` es MD5, `5` es SHA-256, `6` es SHA-512, y `y` es yescrypt (por defecto en Debian moderno).

* **Mina de oro en opasswd**: El archivo `/etc/security/opasswd` almacena hashes de contraseñas previas; al revisarlo, podrías encontrar hashes antiguos en formato MD5 ($1$), que son mucho más fáciles y rápidos de crackear que los actuales en SHA-512 o yescrypt, permitiendo descubrir patrones en la creación de contraseñas del usuario.
