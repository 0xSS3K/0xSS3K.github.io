## Conceptos Clave (TL;DR)

* Microsoft SQL (MSSQL) es un sistema de gestion de bases de datos relacionales de codigo cerrado desarrollado por Microsoft, y fue escrito inicialmente para correr en sistemas operativos Windows.
* Existen versiones que pueden ejecutarse en Linux y MacOS, aunque es mucho mas probable encontrar instancias en objetivos con Windows.
* MSSQL posee bases de datos del sistema por defecto que definen su estructura: master (informacion del sistema), model (plantilla estructural para bases nuevas), msdb (utilizada por SQL Server Agent para programar trabajos) y tempdb (almacena objetos temporales).
* En una instalacion inicial expuesta en red, el servicio SQL suele correr bajo la cuenta NT SERVICE\MSSQLSERVER.

  
## Herramientas Clave

* **SQL Server Management Studio (SSMS):** Aplicacion del lado del cliente instalada frecuentemente por administradores para configuracion y gestion de bases de datos.

* **Impacket's mssqlclient.py:** Cliente en Python muy util para pentesters (preinstalado en muchas distribuciones) para conectarse remotamente e interactuar con la base de datos.

* [Nmap](../../../📂%2008%20Herramientas&Cheatsheets/Nmap.md) Herramienta de escaneo de red que cuenta con scripts predeterminados (ms-sql-*) para atacar el puerto por defecto de MSSQL e identificar versiones e instancias.

* [Metasploit](../../../📂%2008%20Herramientas&Cheatsheets/Metasploit.md) Framework que contiene el escaner auxiliar `mssql_ping` para recopilar informacion vital del servicio en la etapa de footprinting.

  
## Metodología Paso a Paso

1. **Fase de Footprinting:** El objetivo es escanear puertos (por defecto TCP 1433) usando herramientas como Nmap o Metasploit para recopilar informacion valiosa como el nombre del host, la version del software MSSQL, el nombre de la instancia y verificar si los "named pipes" estan habilitados.

2. **Fase de Conexion:** Tras obtener o adivinar credenciales, se procede a la autenticacion remota usando un cliente como `mssqlclient.py`. Esto permite interactuar de forma directa con el motor de base de datos a traves de lenguaje T-SQL.

3. **Fase de Enumeracion Interna:** Una vez conectados al servidor, el primer paso estrategico es listar las bases de datos existentes para comprender el entorno y buscar vectores dentro de las tablas.

## Configuraciones Inseguras

* Clientes MSSQL que no utilizan cifrado para conectarse al servidor.

* Uso de certificados autofirmados (los cuales pueden ser falsificados) cuando el cifrado si esta en uso.

* Habilitacion y uso de "named pipes".

* Credenciales por defecto o debiles para la cuenta `sa` (los administradores suelen olvidar deshabilitar esta cuenta).

  
## Cheat Sheet de Comandos
```bash
# Localizar la ruta del script mssqlclient.py de Impacket en nuestro sistema atacante

locate mssqlclient
```

```bash
# Escaneo agresivo de MSSQL con Nmap: Ejecuta multiples scripts (informacion, contrasenas vacias, cmdshell, configuracion, tablas, hashes), especificando el puerto, credenciales de prueba e instancia.

sudo nmap --script ms-sql-info,ms-sql-empty-password,ms-sql-xp-cmdshell,ms-sql-config,ms-sql-ntlm-info,ms-sql-tables,ms-sql-hasdbaccess,ms-sql-dac,ms-sql-dump-hashes --script-args mssql.instance-port=1433,mssql.username=<USER>,mssql.password=<PASSWORD>,mssql.instance-name=<INSTANCE_NAME> -sV -p 1433 <TARGET_IP>
```

```bash
# Ejecucion del modulo mssql_ping en Metasploit para enumerar detalles del servidor e instancias MSSQL

msfconsole -q -x "use auxiliary/scanner/mssql/mssql_ping; set rhosts <TARGET_IP>; run; exit"
```

```bash
# Conectar al servidor MSSQL remotamente utilizando Autenticacion de Windows mediante la herramienta de Impacket

python3 mssqlclient.py <USER>@<TARGET_IP> -windows-auth
```

```sql
# Comando T-SQL interno (ejecutado dentro del prompt interactivo SQL>) para listar los nombres de todas las bases de datos en el sistema

select name from sys.databases
```


## "Gotchas" y Troubleshooting

* **Credenciales en SSMS:** Debido a que SSMS es una aplicacion cliente, puede estar instalada en cualquier equipo usado para administracion (no solo en el servidor de la BD). Si comprometes la estacion de trabajo de un administrador, puedes encontrar SSMS con credenciales guardadas que te daran acceso a la base de datos.

* **Autenticacion de Windows y Riesgo AD:** Si el servicio utiliza Autenticacion de Windows, el SO procesara el inicio de sesion a traves de su base local SAM o un Controlador de Dominio. Comprometer una cuenta de Active Directory en estos entornos podria facilitar directamente la escalada de privilegios y el movimiento lateral hacia las bases de datos.

* **Falta de Cifrado por Defecto:** Al conectar desde un cliente usando Autenticacion de Windows, el cifrado no es forzado por defecto al intentar la conexion.
