## Conceptos Clave (TL;DR)

* SNMP permite monitorear y administrar dispositivos de red enviando comandos a través del puerto UDP 161.
* El servidor puede enviar paquetes de datos no solicitados al cliente, conocidos como "traps", a través del puerto UDP 162.
* La información se estructura mediante la Management Information Base (MIB), un archivo de texto con una jerarquía de árbol estandarizada.
* Cada nodo en el árbol de la MIB se identifica con un Object Identifier (OID).
* Las versiones SNMPv1 y SNMPv2c no tienen cifrado, por lo que transmiten datos y "Community Strings" (que actúan como contraseñas) en texto plano.
* SNMPv3 aumenta la seguridad introduciendo autenticación con usuario/contraseña y cifrado de transmisión, pero incrementa significativamente la complejidad de configuración.

  
## Herramientas Clave

* **snmpwalk**: Herramienta utilizada para consultar la información almacenada en los OIDs del dispositivo.
* **onesixtyone**: Utilizada para realizar ataques de fuerza bruta rápidos y descubrir los nombres de los community strings.
* **braa**: Permite realizar fuerza bruta sobre OIDs individuales y enumerar su información una vez que se conoce el community string.
* **crunch**: Herramienta complementaria sugerida para crear listas de palabras (wordlists) personalizadas basadas en patrones de nomenclatura del objetivo.

  
## Metodología Paso a Paso

1. **Identificación de Community Strings**: Dado que es común encontrar versiones antiguas de SNMP habilitadas por la complejidad de migrar a v3, el primer paso es descubrir el community string válido utilizando fuerza bruta con `onesixtyone` y diccionarios conocidos.

2. **Extracción de Información (Walk)**: Una vez descubierto un community string válido y si el servicio no requiere autenticación (v1 o v2c), se utiliza `snmpwalk` para consultar el árbol de OIDs y extraer información interna del sistema, como versiones de software o paquetes instalados.

3. **Fuerza Bruta de OIDs (Opcional)**: Si se requiere información de un nodo específico o no se puede volcar todo el árbol, se emplea `braa` junto con el community string descubierto para iterar e identificar la información detrás de OIDs específicos.

  
## Configuraciones Inseguras

Las siguientes configuraciones en el demonio de SNMP representan un riesgo crítico:

* `rwuser noauth`: Proporciona acceso a todo el árbol de OIDs sin requerir ningún tipo de autenticación.

* `rwcommunity <COMMUNITY_STRING> <IPV4_ADDRESS>`: Otorga acceso a todo el árbol de OIDs sin importar desde dónde se enviaron las solicitudes.

* `rwcommunity6 <COMMUNITY_STRING> <IPV6_ADDRESS>`: Proporciona el mismo nivel de acceso que rwcommunity, pero utilizando direcciones IPv6.

* Uso general de SNMPv1 o SNMPv2c, ya que la falta de cifrado permite que los community strings sean interceptados y leídos en la red de forma trivial.

  
## Cheat Sheet de Comandos
```bash
# Revisar la configuración del demonio SNMP local (requiere acceso previo a la máquina)
# Filtra comentarios y líneas vacías para una lectura limpia del archivo de configuración
cat /etc/snmp/snmpd.conf | grep -v "#" | sed -r '/^\s*$/d'
```

```bash
# Fuerza bruta de Community Strings utilizando onesixtyone
# -c especifica el archivo de diccionario (wordlist) para probar contra el objetivo
onesixtyone -c <WORDLIST_PATH> <TARGET_IP>
```

```bash
# Volcar la información del árbol OID usando snmpwalk
# -v2c especifica la versión del protocolo (SNMPv2c)
# -c define el community string descubierto previamente
snmpwalk -v2c -c <COMMUNITY_STRING> <TARGET_IP>
```

```bash
# Fuerza bruta y enumeración de OIDs específicos usando braa
# La sintaxis estricta requiere el formato <community_string>@<IP>:<OID_BASE>.*
braa <COMMUNITY_STRING>@<TARGET_IP>:.1.3.6.*
```


## "Gotchas" y Troubleshooting
* Muchos administradores asocian los community strings a direcciones IP específicas y utilizan el nombre de host del equipo u otros patrones como base para el nombre del string, en ocasiones añadiendo símbolos.
* Si las listas de palabras estándar (como las de SecLists) fallan en la identificación, considera usar herramientas como `crunch` para generar wordlists personalizadas si identificas un patrón de nomenclatura en la red objetivo.
* La falta de cifrado en las versiones 1 y 2c significa que, si estás posicionado en la red interceptando tráfico, el community string puede ser capturado y leído cada vez que se envía a través de la red.
