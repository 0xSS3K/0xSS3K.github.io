## Conceptos Clave (TL;DR)

* Server Message Block (SMB) es un protocolo cliente-servidor para acceder a archivos, directorios y recursos de red compartidos.
* Samba es la implementación alternativa (de código abierto) de SMB que permite la interoperabilidad entre distribuciones Linux/Unix y sistemas Windows.
* El acceso a los recursos se gestiona mediante Listas de Control de Acceso (ACLs) vinculadas directamente a los recursos compartidos (shares), que son independientes de los permisos locales asignados en el servidor.
* Las versiones antiguas (CIFS/SMB1) operan sobre NetBIOS (TCP 137, 138, 139), mientras que las implementaciones modernas operan de forma exclusiva sobre conexión directa TCP 445.

### Herramientas Clave

* **smbclient**: Cliente interactivo en línea de comandos para listar y conectarse a recursos compartidos SMB.
* **smbstatus**: Herramienta administrativa local para verificar conexiones activas, versiones de protocolo y usuarios en un servidor Samba.
* [**Nmap**](../../../%F0%9F%93%81%2001_Reconocimiento_y_Enumeracion/Footprinting/Host%20Based%20Enumeration/brain-not-braining/%F0%9F%93%82%2008_Herramientas_y_Cheatsheets/Nmap.md): Escaneo inicial de puertos y detección de versiones mediante scripts NSE (aunque puede ser limitado y lento para SMB).
* **rpcclient**: Herramienta de MS-RPC; excelente para enumerar políticas, usuarios, grupos y recursos compartidos mediante sesiones nulas.
* **samrdump.py (Impacket)**: Script en Python para volcar rápidamente información detallada de usuarios del dominio.
* **smbmap &** [**NetExec**](../../../%F0%9F%93%81%2001_Reconocimiento_y_Enumeracion/Footprinting/Host%20Based%20Enumeration/NetExec.md): Herramientas de automatización para mapear recursos compartidos y verificar permisos de acceso masivamente.
* **enum4linux-ng**: Herramienta de enumeración automatizada exhaustiva (políticas de contraseñas, sesiones nulas, usuarios, grupos, shares) basada en múltiples técnicas subyacentes.

### Metodología Paso a Paso

1. **Reconocimiento Inicial y Escaneo**: Identificar si los puertos 139 y 445 están abiertos y qué versiones de SMB se están ejecutando en el objetivo.
2. **Mapeo de Recursos Compartidos (Null Session)**: Intentar listar los recursos disponibles utilizando sesiones anónimas (sin proveer contraseña) para descubrir nombres de carpetas interesantes y permisos iniciales.
3. **Exploración de Recursos**: Conectarse a los recursos compartidos accesibles para inspeccionar y descargar archivos de valor (archivos de configuración, scripts, notas).
4. **Enumeración a través de RPC**: Utilizar `rpcclient` interactivo con sesión nula para extraer información profunda de red: configuración del dominio, políticas de contraseñas, usuarios y grupos.
5. **Fuerza Bruta de RIDs**: Si las consultas directas y generales están bloqueadas, iterar manualmente sobre los Identificadores Relativos (RIDs) para descubrir cuentas de usuario que de otro modo permanecerían ocultas.
6. **Verificación Automatizada**: Usar múltiples herramientas (CrackMapExec, SMBMap, Enum4linux-ng) para confirmar hallazgos y descubrir vectores adicionales, ya que diferentes herramientas pueden interpretar los paquetes de manera distinta.

### Cheat Sheet de Comandos

```bash
# Filtrar comentarios para revisar rápidamente la configuración local de Samba (si ya se tiene acceso por terminal)
cat /etc/samba/smb.conf | grep -v "#\|\;"
 

# Reiniciar el servicio Samba localmente después de aplicar/modificar configuraciones
sudo systemctl restart smbd

  
# Escaneo de Nmap enfocado en puertos SMB con scripts por defecto y detección de versiones
sudo nmap <TARGET_IP> -sV -sC -p139,445
  

# Listar recursos compartidos (shares) usando una sesión nula (acceso anónimo, sin contraseña)
smbclient -N -L //<TARGET_IP>
  

# Conectarse interactivamente a un recurso compartido específico (acepta contraseña vacía para anónimo)
smbclient //<TARGET_IP>/<SHARE_NAME>
  

# COMANDOS INTERNOS DE SMBCLIENT (Una vez establecida la sesión):
get <FILE_NAME>    : Descargar un archivo al host local.

!<CMD>             : Ejecutar comandos del sistema local (ej. !ls, !cat) sin cerrar la sesión SMB actual.

help               : Listar todos los comandos disponibles dentro de la sesión interactiva.

  

# Verificar quién está conectado a los recursos compartidos, IPs de origen y proceso (requiere acceso shell al servidor)
smbstatus

  

# Iniciar sesión MS-RPC usando una sesión nula
rpcclient -U "" <TARGET_IP>


# COMANDOS INTERNOS DE RPCCLIENT (Dentro de la consola rpcclient $>):

srvinfo               : Obtener información del servidor (SO, plataforma, versión).

enumdomains           : Enumerar dominios desplegados en la red.

querydominfo          : Extraer información básica del dominio, servidor y cantidad de usuarios.

netshareenumall       : Listar todos los recursos compartidos disponibles.

netsharegetinfo <SHARE> : Obtener información técnica y configuración de un share específico.

enumdomusers          : Listar todos los usuarios del dominio.

queryuser <RID>       : Obtener información sumamente detallada de un usuario específico.

querygroup <RID>      : Obtener información descriptiva de un grupo y número de miembros.
 

# Fuerza bruta de RIDs (ejemplo del 500 al 1100) vía rpcclient para descubrir nombres de usuarios y sus grupos
for i in $(seq 500 1100);do rpcclient -N -U "" <TARGET_IP> -c "queryuser 0x$(printf '%x\n' $i)" | grep "User Name\|user_rid\|group_rid" && echo "";done


# Volcar información de usuarios y configuraciones en bloque usando el script de Impacket
samrdump.py <TARGET_IP>


# Mapear rápidamente recursos compartidos y revelar los niveles de permisos de acceso (lectura/escritura)
smbmap -H <TARGET_IP>
 

# Enumerar shares comprobando acceso con credenciales vacías (sesión nula)
crackmapexec smb <TARGET_IP> --shares -u '' -p ''

  
# Ejecutar enumeración integral agresiva y automatizada (LDAP, NetBIOS, SMB, RPC, Políticas de contraseñas, Sesiones)
./enum4linux-ng.py <TARGET_IP> -A
```

### "Gotchas" y Troubleshooting

* **Limitaciones de Escáneres Automatizados**: Nmap puede tardar mucho tiempo en escanear SMB y a menudo se salta detalles cruciales de las ACL y recursos. Siempre es imperativo complementar con enumeración manual minuciosa (mediante `rpcclient` y `smbclient`).
* **Discrepancia entre Herramientas**: Nunca confíes en una sola herramienta automatizada. Debido a diferencias de programación en su motor subyacente, herramientas como enum4linux, smbmap y crackmapexec pueden recuperar datos conflictivos o parciales. Hay que triangular la información obtenida.
* **Restricciones de RPCclient**: Es bastante común que comandos directos como `enumdomusers` arrojen un error (Access Denied) por restricciones en usuarios anónimos. Sin embargo, la consulta granular `queryuser <RID>` casi siempre funciona, por lo que la técnica del bucle `for` para iterar RIDs es fundamental.
* **Disponibilidad de Puertos y Dialectos**: Los entornos antiguos dependen de NetBIOS sobre TCP (puertos 137-139). CIFS (SMB1 puro) y las versiones modernas (SMB 2.0/3.0) prefieren conexión directa al puerto 445. Enumerar siempre 139 y 445 en conjunto.

### Configuraciones Inseguras

\| `browseable = yes` | `/etc/samba/smb.conf` (Nivel Share)  | Permite que el recurso compartido sea visible y listable en la estructura de red, entregando a los atacantes una estructura pre-mapeada. |

\| `read only = no` / `writable = yes` | `/etc/samba/smb.conf` (Nivel Share)  | Permite que los usuarios conectados no solo lean, sino que creen, editen o reemplacen archivos; vector crítico de explotación. |

\| `guest ok = yes` / `usershare allow guests = yes` | `/etc/samba/smb.conf` (Nivel Global o Share)  | Permite conexiones anónimas sin requerir contraseña, exponiendo todos los recursos contenidos bajo dicha política. |

\| `map to guest = bad user` | `/etc/samba/smb.conf` (Nivel Global)  | Si se ingresa un usuario inexistente (bad user), Samba fuerza la conexión mapeándola hacia la cuenta local "guest", evadiendo controles de login fallido. |

\| `create mask = 0777` / `directory mask = 0777` | `/etc/samba/smb.conf` (Nivel Share)  | Define la creación de cualquier archivo o carpeta nueva con permisos máximos y absolutos (lectura/escritura/ejecución para todos en el sistema). |

\| `enable privileges = yes` | `/etc/samba/smb.conf` (Nivel Share)  | Permite la aplicación directa de privilegios críticos en función de SIDs, lo cual puede ser escalable si existen configuraciones de grupo laxas. |

\| `logon script = <script.sh>` / `magic script = <script.sh>` | `/etc/samba/smb.conf` (Nivel Global o Share)  | Ejecuta archivos en el sistema local al conectarse/desconectarse. Si la carpeta es escribible, el atacante sobreescribe el script, obteniendo RCE de manera trivial. |
