## Conceptos Clave (TL;DR)

* [IMAP](../../../%F0%9F%93%81%2001_Reconocimiento_y_Enumeracion/Footprinting/Host%20Based%20Enumeration/ServiceEnum/) (Internet Message Access Protocol) permite gestionar correos de forma online y estructurada (carpetas) directamente en el servidor, manteniéndolos ahí hasta su borrado y actuando como un sistema de archivos de red para clientes.
* [POP3](../../../%F0%9F%93%81%2001_Reconocimiento_y_Enumeracion/Footprinting/Host%20Based%20Enumeration/ServiceEnum/)(Post Office Protocol) posee funcionalidades más limitadas; se enfoca en listar, recuperar y eliminar correos del servidor, sin soporte para estructuras de carpetas nativas.
* La comunicación estándar de estos protocolos se realiza enviando comandos de texto en formato ASCII.
* De forma predeterminada los protocolos operan sin cifrar, enviando credenciales y correos en texto claro; se recomienda y utiliza SSL/TLS en puertos alternativos para asegurar el tráfico.

### Herramientas Clave

* [Nmap](../../../%F0%9F%93%81%2001_Reconocimiento_y_Enumeracion/Footprinting/Host%20Based%20Enumeration/brain-not-braining/%F0%9F%93%82%2008_Herramientas_y_Cheatsheets/Nmap.md): Utilizado para descubrimiento de puertos, banner grabbing, enumeración de capacidades de los servicios y extracción de información de certificados SSL.
* **cURL**: Permite la interacción directa mediante línea de comandos para autenticarse, listar directorios y depurar la conexión (viendo detalles del handshake TLS y versiones).
* **OpenSSL (`s_client`)**: Imprescindible para establecer túneles de conexión manuales e interactuar con IMAP/POP3 cuando estos operan sobre puertos seguros (TLS/SSL).

### Metodología Paso a Paso

1. **Descubrimiento y Escaneo**: Ejecutar Nmap sobre los puertos por defecto de IMAP y POP3 (texto claro y seguros). El objetivo es identificar qué servicios están activos, enumerar sus capacidades (comandos soportados) y extraer metadatos organizacionales de los certificados SSL.
2. **Conexión e Interacción (TLS/SSL)**: Debido a que muchos servidores fuerzan conexiones cifradas, utilizar OpenSSL o cURL para eludir problemas de cifrado y establecer un canal directo con el servicio.
3. **Autenticación y Enumeración Interna**: Usando credenciales obtenidas (por reutilización, fuerza bruta o valores predeterminados), iniciar sesión en el servicio para leer correos, inspeccionar la estructura de directorios y buscar información sensible o adjuntos de valor.

### Configuraciones Inseguras

Errores en la configuración del servidor de correo pueden exponer la plataforma a fugas de información, permitiendo a un atacante leer correos o iniciar sesión sin credenciales válidas.

* **Inicio de Sesión Anónimo**: Similar a FTP, un servidor mal configurado puede permitir autenticación anónima. `auth_anonymous_username` define el usuario utilizado para la autenticación bajo el mecanismo ANONYMOUS SASL.
* **Fuga de Credenciales y Verbosidad en Logs**:

&#x20; \* `auth_debug`: Habilita todo el registro de depuración del proceso de autenticación.

&#x20; \* `auth_debug_passwords`: Registra en los logs las contraseñas enviadas y el esquema utilizado.

&#x20; \* `auth_verbose`: Registra los intentos de autenticación fallidos detallando las razones.

&#x20; \* `auth_verbose_passwords`: Permite registrar las contraseñas usadas para la autenticación (aunque pueden aparecer truncadas).

### Cheat Sheet de Comandos

#### Reconocimiento y Conexión

```bash
# Escaneo de puertos estándar y seguros de IMAP/POP3, enumeración de versiones y ejecución de scripts por defecto
sudo nmap <TARGET_IP> -sV -p110,143,993,995 -sC

  
# Interacción y autenticación IMAPS vía cURL ignorando errores de certificado (-k)
curl -k 'imaps://<TARGET_IP>' --user <USER>:<PASSWORD>

  
# Interacción IMAPS vía cURL con máxima verbosidad (-v) para leer información del certificado SSL, versión TLS y banners
curl -k 'imaps://<TARGET_IP>' --user <USER>:<PASSWORD> -v
  

# Conexión interactiva cifrada vía OpenSSL a POP3 seguro (puerto 995 por defecto, aunque en el módulo se indica pop3s)
openssl s_client -connect <TARGET_IP>:pop3s

  
# Conexión interactiva cifrada vía OpenSSL a IMAP seguro (puerto 993 por defecto)
openssl s_client -connect <TARGET_IP>:imaps
```

#### Interacción Directa - Comandos IMAP

_Nota: Los comandos IMAP requieren un identificador alfanumérico al inicio (ej. "1", "A001") para emparejar la solicitud con la respuesta del servidor._

```
# Autentica al usuario en el servidor IMAP
1 LOGIN <USER> <PASSWORD>

  
# Lista todos los directorios del buzón de correo
1 LIST "" *

  
# Crea un buzón de correo con el nombre especificado
1 CREATE "INBOX"

  
# Elimina el buzón de correo especificado
1 DELETE "INBOX"

  
# Renombra un buzón de correo
1 RENAME "ToRead" "Important"

  
# Devuelve un subconjunto de buzones activos/suscritos por el usuario
1 LSUB "" *

  
# Selecciona un buzón para acceder y leer los mensajes contenidos
1 SELECT INBOX

  
# Sale del buzón actualmente seleccionado
1 UNSELECT INBOX
 

# Recupera los datos de un mensaje específico basado en su ID
1 FETCH <ID> all

  
# Elimina definitivamente todos los mensajes marcados para borrado (Deleted flag)
1 CLOSE


# Cierra la conexión con el servidor IMAP
1 LOGOUT
```

#### Interacción Directa - Comandos POP3

```
# Identifica al usuario para el inicio de sesión
USER <USER>
  

# Envía la contraseña correspondiente para autenticación
PASS <PASSWORD>

  
# Solicita el número total de correos electrónicos guardados
STAT

  
# Solicita el número y el tamaño de todos los correos electrónicos
LIST

  
# Solicita al servidor que entregue un correo específico por su ID
RETR <ID>

  
# Solicita al servidor que elimine un correo específico por su ID
DELE <ID>

  
# Solicita al servidor que muestre las capacidades (capabilities) habilitadas
CAPA

  
# Solicita al servidor que restablezca/reinicie la información transmitida (aborta cambios pendientes)
RSET

  
# Cierra la conexión con el servidor POP3
QUIT
```

### "Gotchas" y Troubleshooting

* **Puertos por Defecto**: POP3 opera sobre el puerto 110 (texto claro) y 995 (TLS/SSL). IMAP opera en el puerto 143 (texto claro) y 993 (TLS/SSL).
* **Peculiaridad de Comandos IMAP**: A diferencia de otros protocolos, IMAP permite enviar múltiples comandos sin esperar la confirmación del servidor. Para organizar las respuestas, cada comando debe ir precedido de un identificador (ej. `1 LOGIN`, `A001 CAPABILITY`), que el servidor incluirá en su respuesta.
* **Verificación de Certificados**: En muchas evaluaciones los certificados SSL/TLS son autofirmados ("self signed certificate"), lo que romperá las conexiones de clientes por defecto. Utiliza la bandera `-k` en `curl` para omitir esta validación.
* **Modo Offline en Clientes**: Algunos clientes de correo mantienen una caché local u offline; los cambios realizados ahí se sincronizan con el servidor y otros clientes automáticamente una vez que se restablece el puerto de comunicación, lo cual es exclusivo de la arquitectura IMAP.
